# Сайт с отправкой заявок в Telegram

## Что внутри
- `public/index.html` — сайт
- `server.js` — backend, который принимает форму и шлёт заявку в Telegram
- `.env.example` — пример переменных окружения

## Что нужно сделать
1. Создать Telegram-бота через `@BotFather`
2. Получить `TELEGRAM_BOT_TOKEN`
3. Узнать `TELEGRAM_CHAT_ID`, куда должны приходить заявки
4. Скопировать `.env.example` в `.env` и заполнить значения
5. Установить зависимости:
   `npm install`
6. Запустить:
   `npm start`

## Как получить chat_id
Самый простой путь:
- написать вашему боту любое сообщение
- открыть в браузере:
  `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
- найти `chat.id`

## Как это работает
Форма "Получить расчет" отправляет POST-запрос на `/api/lead`.
Backend получает поля:
- имя
- телефон
- адрес
- комментарий

И сразу отправляет их в ваш Telegram через Bot API.

## Важно
Без `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` автоматическая отправка работать не будет.


Уже заполнено:
- TELEGRAM_BOT_TOKEN

Осталось заполнить только:
- TELEGRAM_CHAT_ID

Как получить TELEGRAM_CHAT_ID:
1. Напишите вашему боту любое сообщение в Telegram.
2. Откройте в браузере:
   https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
3. Найдите поле chat -> id
4. Вставьте это число в файл .env вместо 'вставить_chat_id'


Формат заявок обновлён: бот отправляет красиво оформленное сообщение с иконками и полями.


НОВОЕ В ЭТОЙ ВЕРСИИ
- Быстрая заявка на первом экране
- Красивое сообщение в Telegram
- Время заявки
- Источник заявки
- Тип формы (основная или быстрая)
- Кликабельный телефон в сообщении
- Локальный журнал заявок в файле leads.jsonl

КУДА СОХРАНЯЮТСЯ ЗАЯВКИ
- В Telegram
- В файл leads.jsonl в корне проекта

КАК СМОТРЕТЬ ЛОКАЛЬНЫЕ ЗАЯВКИ
Откройте файл:
leads.jsonl

Каждая строка — это отдельная заявка в формате JSON.
